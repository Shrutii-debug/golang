1. Package Declaration
package main
What it means

This file belongs to the main package.

Intuition

In Go, programs are organized into packages.
The main package is special because:

Go compiler looks for main.main()

and starts execution there.

So:

package main → executable program

3. Course Struct (Model)
type Course struct{
	CourseId string `json:"courseid"`
	CourseName string `json:"coursename"`
	CoursePrice int `json:"price"`
	Author *Author `json:"author"`
}
Intuition

A struct represents a data model.

Here you are modelling a course entity.

Example:

Course
 ├── id
 ├── name
 ├── price
 └── author

Equivalent JSON:

{
  "courseid": "2",
  "coursename": "ReactJS",
  "price": 299,
  "author": {}
}
Struct tags
`json:"courseid"`

Intuition:

Go field name → JSON field name

Without this:

CourseId → CourseId

With tag:

CourseId → courseid

This matches frontend expectations.

4. Pointer Author
Author *Author
Intuition

This means:

Course → contains reference to Author

Instead of copying Author object.

Memory idea:

Course
  |
  → pointer → Author

Advantages:

avoids copying data

optional field possible

flexible structure
